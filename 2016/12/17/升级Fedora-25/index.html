升级了Fedora-25

发表于 2016-12-17 | 更新于 2016-12-16 | 分类于 操作系统 |

0x00 背景

系统推送了Fedora 25的升级，看了下Release Notes，大体都是更新升级，没有太大的变化，内核更新，Gnome升级，etc.

  • Release Notes - Fedora 25
  • Installation Guide - Fedora 25

主要看了下 for Developer部分，Ruby，Go，Erlang等都进行了版本的更新，另外增加了Unicode 9.0的支持，以为了增加了更多的字符和奇怪的表情～
so 就升级了。

0x01 升级过程

现在fedora的升级，从23开始，就比过去简单了很多，然后成功率也提高了不少😆.
升级方法主要有dnf升级，安装升级2种。

01.dnf升级

我用的dnf升级，主要是因为懒。
整体耗时大约1小时左右，整体更新文件大约3000左右，可以接受。
具体的升级流程很简单：

  1. dnf upgrade //本地的应用更新到最新
  2. dnf install dnf-plugin-system-upgrade //安装dnf系统升级组件
  3. dnf system-upgrade download –refresh –releasever=25 //下载25
  4. dnf system-upgrade reboot //执行重启，安装

下载的数据量，会根据你安装的应用数量不同，我系统前几天刚刚把没用的应用都干掉了，干净的很，因此只下载了2个G左右，这里面包括系统本身1个多G，还有mysql，gstream，emacs等等大家伙。
升级过程中，没有大的问题，刚开始的时候，chrome因为众所周知的原因，更新很慢，还卡断了2次，因此直接卸了。其实mysql的更新速度也不咋滴，时间就是金钱，应该也卸了，能大大提升速度。

02.重新安装

另外一种方法，就是下载ISO，覆盖安装。
没有实验，还要ISO启动，太麻烦。

0x02 更新测试

01.更新的内容

  1. gnome升级了，但没有大的感觉。//默认壁纸换了，但是感觉没有24的好看，右下的fedora也很模糊。
  2. Display Server默认改为Wayland。// 没啥感觉，之前也用，X11已经很多年不用了
  3. 整合了MP3编解码。//可能对新装系统还有点用，对于升级的，然并卵
  4. Kernel升级，增加Media Writer，Flappack，etc. //对于我都没啥大的意义
  5. unicode9.0升级，ibus增加emoji输入。//唯一感觉有意义的两个，再也不用查unicode了，👳👯🅰👭🍖🍌，不过ibus的intellege pinyin还不支持，等待更新。

02.基本应用

  1. 官方源的，大部分都变成25的了，没怎么仔细看，只是注意到setting里面部分样式变了，比如快捷键设置改为单页了，取消了tab。
  2. 常用软件libra，gimp，没有变化，正常使用。
  3. firefox，thunderbird，正常使用 。
  4. 部分第三方源的，存在没更新的，比如我通过fzug装了个网易云音乐，暂时还是24的，但是不影响使用。

03.开发环境

  1. docker还是24的，估计要等docker官方的repo升级了。
  2. java，node，ruby，hsakell，这几个我在用的，不需要任何操作，都能正常工作。
  3. eclipse，idea系列正常工作。
  4. git，maven正常工作。
  5. mysql，workbench正常工作。
  6. atom挂了，目前唯一挂掉的应用，报错信息 not a valid Electron app.
    atom现在没有分发和自动升级，看了下，报错的版本是1.10，忘了是从哪装的了，卸载了后，从github get了最新的1.12，重装问题解决。（注意不要删掉配置和插件，这样重装之后就不用重新配置了）

0x03 总结

目前情况，基本这样，是本着用新不用旧的原则，进行了升级，而且作为开发环境，本地也没有特别重要的数据，只是稍微备份了一下部分软件的配置文件，和本地maven repo。
暂时也没看出来优点，缺点，用用再说吧，随时更新本文。
