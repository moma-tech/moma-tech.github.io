使用Github，Hexo搭建个人Blog

发表于 2016-12-09 | 更新于 2016-12-16 | 分类于 互联网服务工具 |

因为心水hexo漂亮简单的主题NexT，所以用github搭了一套。

主要基于：

  • Github
  • Hexo Blog Framework
  • NexT 主题

以下的安装配置基于本人的Fedora，其他系统的大同小异

0x01 环境准备

01.本地Node.js安装

Hexo系统使用node.js开发，因此需要本地预先安装node.js.
Fedora直接使用 dnf 安装即可，没有的请检查下自己的源.

安装成功的，可以顺利执行npm命令.
失败的，可能没洗手，去洗手，然后重新来一次.

想追新的，可以去编译 源码，但对于本次内容，然并卵

02.本地GIT安装

因为使用Github作为页面空间，所以需要安装Git.
Fedora默认已经安装了，没有的dnf一下，注意下包名（git-core） ，同样有问题的，洗手，检查自己的源.

没用过git的没有关系，本次内容用不到git命令，想了解的，可以各种搜索.
此处有推荐 : 《Pro Git简体中文》.

0x02 Hexo网站搭建

先上官方英文教程 hexo Docs

01.安装Hexo Cli

使用npm 安装 hexo cli.

安装成功的，会成功输出hexo-cli版本在内的相关信息，有问题的，你知道怎么做的.

02.网站初始化

hexo安装成功后，就可以进行网站的搭建了.
首先选择合适的目录，然后执行初始化命令.
本人此处选择的目录是 projects/hexo-site

安装成功后，基础目录就自动创建完成了.

03.本地测试

网站文件夹建立完成后，可以通过hexo cli命令，启动本地服务，进行查看.

启动后，浏览器 localhost:4000 可以顺利看到站点.

此处clean命令用于清除缓存，建议后续有配置文件的修改后，都clean一下.

0x03 NexT主题安装

01.NexT下载安装

先上官方中文教程 NexT 开始使用

NexT的安装，实际就是把NexT相关文件放到Hexo站点的theme目录下面，官方推荐的方法是 git clone.
当然下载，解压，也是同样可以的.

02.NexT主题启用

主题安装完成后，需要在站点配置文件(_config.yml)启用.

_config.yml很重要，详细说明请参考 Hexo Configuration
具体步骤：

  1. 使用任意editor打开 hexo-site_config.yml，此处有推荐 EMACS.
  2. 先修改开始几行的 language: zh-Hans，这样主题会使用简体中文.
  3. CTL-F 查找 theme 关键字，应该是在Extensions部分，直接滚也行，没几行.
  4. 修改 theme： next
  5. 保存，退出.
    站点配置文件的前几行，是网站的一些基本信息，添加一下，简单英文，看不懂的请洗手查字典.
    部分参考：

    language: zh-Hans
    timezone: UTC

0x04 创建文章

01.创建POST

执行hexo new命令，可以在source/_post目录下自动创建文章

02.编辑内容

Post使用makedown格式，由两部分组成.
头部为yaml/json格式的属性，用三条短横线/三个分号与后续正文分隔.
YAML

JSON

头部支持的属性，请参考 Hexo Front Matter

正文部分，使用Markdown格式，基础的md标准语法都能支持，不太会的，直接写html也可以的.
没用过md的，请自行洗手百度.

0x05 Hexo网站发布

01.Github Repo创建

先上官方英文教程 Github Page 首页往下拉

  1. 首先你要有个github账号
  2. 点击 New Repository
  3. repo的名字为 (你的username).github.io

    参考图：此处我的username为moma-tech，repo名为moma-tech.github.io
    PS: username的修改在 setting – Account – Change username

02.网站配置

此处主要修改站点配置文件，增加github的发布信息.
步骤如下：

  1. 使用任意editor打开 hexo-site_config.yml，此处有推荐 EMACS.
  2. 首先修改URL部分的 url: https://moma-tech.github.io
  3. 查找Deployment部分

  4. deploy修改如下

  5. 保存，退出.

03.网站发布

  1. 首先安装 hexo 的 git deply 插件

  2. 执行 clean， generate 命令

  3. 执行 deploy 命令

    [ivan@ivan hexo-site]$ hexo deploy

    过程中需要输入 github的用户名及密码，不想每次都输入的，可以配置github的SSH，具体请洗手百度.

  4. 提示发布完成后，浏览器输入 (你的username).github.io 查看
