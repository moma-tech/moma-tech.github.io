0x01 理查德森成熟模型

REST 是由 DR.Roy Fielding于2000年他的博士论文中提出的，完全基于HTTP协议。当时他更多的像是一种通信风格，而不是一种标准。
REST 不是一种要么完全选择要么就放弃的技术，通常，用户更多选择去包含REST中的部分，去更有效率的配合他们的场景。
目前APIs经常会被称为“RESTful”，当然这要根据它们对REST原则的遵守情况。
我们可以使用理查德森成熟模型来对REST APIs进行分类辨别。
Matain Fowler针对这个模型有过一次详细的介绍，Expaining the concepts，下面是一个简短的版本：

  • Level 0 The Swamp of POX
    当使用HTTP纯粹作为通信手段的时候，API通常会处于这个此等级。例如，目前很多的SOAP服务都是基于HTTP，实际完全可以通过JMS或者SMTP实现。这一类的服务通常我们认为式最初级的RESTFUL。POX指的是老式的文本XML（Plain Old Xml），同时大部分处于此级别的技术，比如SOAP、RPC-XML，使用HTTP传输使用XML编码的数据信息，通常基于远程过程调用的语义模型，一般是SCHEMA。
  • Level 1 Resources
    在这个级别，一个单体服务节点可以被扩展到多个，用于区分HTTP资源，通常一个资源对应系统中的一个对象，比如一条消费记录，或者一个用户记录。通常这些资源的相关联了其他资源，表现在他的拓扑URI表述中。比如，你可以暴露你系统中的每单个资源，例如 /user/1，/user/2.
  • Level 2 Transpost Native Properties
    在这个级别，API的设计构建使用了HTTP中的特殊声明定义，例如头信息（headers），状态码（status code），以及动词（get，post，delete…）。这些构成了一个更为标准的HTTP API，同时也只有这个级别才会被认为初步使用了REST框架设计。
  • Level 3 Hypermedia
    这个终极级别，主要是基于HATEOAS（HyperText As The Engine Of Application State)。他包括API与URI拓扑之间的解耦，以及通过响应中的元数据信息向客户端暴露可用的浏览选项。该元数据信息通常使用包装结构来传输，通常为一系列的链接，同时该信息能够被正确表达和理解。

通常大部分的REST框架，可以帮助你处于level 2，例如使用Spring MVC，默认便是此级别。

此处补充下HATEOAS，未使用HATEOAS，客户端依然需要服务端提供相关的文档，以了解暴露的资源和对应的操作。同时当服务端发生变化，比如修改了URI，客户端需要作对应的修改。而使用HATEOAS，所有的资源都将通过响应来表达，客户端可以基于此表达，智能发现可行操作。当服务端发生变化时，客户端不受任何影响，因为URI和其他信息都是动态发现的。

0x03 The Web Application

Web应用，主要指我们的REST API是一个符合Servlet-3标准的Spring MVC应用。这里我们使用Servlet 3工具集来通过程序引导相关的容器，对应的是传统的使用web.xml。

此处记录下，Spring Boot支持多种容器引导形式，可以用于取代传统的web.xml，常见的有使用WebApplicationInitializer，以及这里的AbstractAnnotationConfigDispatcherServletInitializer，但是大多数情况下，可以直接使用SpringBootServletInitializer

CrmWebApplicationInitializer主要配置了一个SpringMVC应用的标准2层结构。

  • web-application-global Spring contexts，Spring的全局上下文配置。
  • Template methods，用于注册Servlet的类型结构，例如Filter实例，同时自定义其注册形式。

另外一个WebMvcConfiguration使用了EnableWebMvc注解，同时指定了Spring’s的MultipartResolver使用Servlet 3中的实现。这个实现用来明确Spring如何对待文件上传。

其中@EnableWebMvc注解，其提供了一系列的功能。
在这里，Spring MVC将会自动完成：

  • 支持Classpath中声明的 JAXB 实现，用于在REST服务中进行object-to-xml的编译
  • 支持Classpath中声明的校验器，如Hibernate Validator，用于使用JSR 303标准注解校验请求中的信息内容
  • 支持通过ROMA LIB，渲染RSS/ATOM 消息
  • 支持将使用@Controller的bean暴露为一个HTTP服务节点
